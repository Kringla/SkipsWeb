MANUELLE TESTER
Under følger en punkt-for-punkt-sjekk med forventet utfall. Gjøres i første omgang med Chrome.

Nr	Side							Handling / input								Forventet resultat
A1	/register.php					Registrer ny bruker (ADM+USR)					Skjema valideres, ny rad i users-tabellen, redirect til login.php
A2	/login.php		 				Logg inn som ADM / USR							ADM → dashboard.php, USR → fartøy-søk (eller dashboard)
A3	/logout.php						Klikk “Logg ut”									Sesjon ødelagt, redirect til login.php
B1	/dashboard.php					Åpnes uten innlogging							Skal redirecte til login.php
B2	/dashboard.php (ADM)			Åpnes som ADM									Viser meny (inkl. lenke til søk og senere CRUD)
B3	/dashboard.php (USR)			Åpnes som USR									Viser meny uten CRUD-lenker
C1	/user/fartoy_sok.php			Tomt søk										Viser tomt resultat eller hjelpetekst
C2	/user/fartoy_sok_form.php		Søk på eksakt båt-ID eller delstreng i navn		Viser treff-tabell med korrekt kolonner og data
C3	/user/fartoy_sok_results.php	Klikk på båt-rad								Viser detaljer (navnehistorikk, spesifikasjoner)
D1	Feilhåndtering					Slå av DB-tilgang (skriv feil passord)			Får veldefinert “Internal server error” (500) og logg
D2	Ugyldige parameter				Send script uten ?id=… eller med ?id=99999		Viser “Ingen data” eller “Invalid ID” uten PHP-error
